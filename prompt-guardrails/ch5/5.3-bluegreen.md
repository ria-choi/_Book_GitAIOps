# 5.3 Blue/Green 배포

## 실행 지침

### 단계 1: Argo Rollouts 설치

```bash
kubectl create namespace argo-rollouts
kubectl apply -n argo-rollouts -f https://github.com/argoproj/argo-rollouts/releases/latest/download/install.yaml
```

kubectl 플러그인 설치:
```bash
brew install argoproj/tap/kubectl-argo-rollouts  # macOS
```

### 단계 2: Deployment → Rollout 전환

기존 `k8s/smb/deployment.yaml`을 삭제하고 `k8s/smb/rollout.yaml`을 생성한다:

- `apiVersion: argoproj.io/v1alpha1`, `kind: Rollout`
- Blue/Green 전략:
  ```yaml
  strategy:
    blueGreen:
      activeService: notiflex-api
      previewService: notiflex-api-preview
      autoPromotionEnabled: true
      autoPromotionSeconds: 30
  ```
- preview 서비스도 생성 (`k8s/smb/service-preview.yaml`)

> ⚠️ **중요**: deployment.yaml과 rollout.yaml이 동시에 존재하면 ArgoCD가 양쪽 모두 배포한다. 반드시 deployment.yaml을 삭제한다.

> ⚠️ **전략 변경 참고**: 이후 6장에서 Blue/Green → Canary로 변경할 때, `kubectl apply`만으로는 전환되지 않을 수 있다. Rollout을 삭제하고 다시 생성해야 한다.

### 단계 3: Blue/Green 배포 테스트

앱 버전을 올리고 배포한다:
```bash
gcloud builds submit app/ --tag=asia-northeast3-docker.pkg.dev/<PROJECT_ID>/notiflex/api:v0.2.0
```

Rollout 이미지 태그를 v0.2.0으로 변경, git push 후 ArgoCD sync를 확인한다.

```bash
kubectl argo rollouts get rollout notiflex-api -n notiflex -w
```

30초 후 auto-promote되어 active로 전환되는 것을 확인한다.

### 단계 4: 커밋

변경 사항을 커밋하고 푸시한다.
