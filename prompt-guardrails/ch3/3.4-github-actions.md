# 3.4 GitHub Actions CI

## 실행 지침

### 단계 1: CI 파이프라인 생성

`.github/workflows/ci.yaml` 파일을 생성한다:

- **트리거**: main 브랜치에 push (app/ 디렉터리 변경 시)
- **단계**:
  1. 코드 체크아웃
  2. GCP 인증 (Workload Identity 또는 Service Account Key)
  3. Docker 이미지 빌드
  4. Artifact Registry에 푸시
  5. 이미지 태그: git SHA 기반

### 단계 2: GCP 인증 설정

GitHub Secrets에 GCP 서비스 계정 키를 등록한다:
- `GCP_PROJECT_ID`
- `GCP_SA_KEY` (JSON)
