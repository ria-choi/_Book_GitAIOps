# 7.3 App of Apps 패턴

## 실행 지침

### 단계 1: App of Apps 구조 생성

`argocd/root-app.yaml` — 루트 Application:
```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: root-app
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://github.com/<USER>/notiflex-platform.git
    targetRevision: main
    path: argocd
    directory:
      recurse: true
  destination:
    server: https://kubernetes.default.svc
    namespace: argocd
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
```

`argocd/` 디렉터리에 각 Application 매니페스트를 배치한다. 루트 Application이 이 디렉터리를 감시하여 하위 Application들을 자동 관리한다.

> ⚠️ **directory.recurse 필수**: `argocd/` 하위에 `smb/`, `enterprise/` 등 서브디렉터리를 사용하면 `directory.recurse: true`를 설정해야 한다. 이 옵션이 없으면 ArgoCD가 루트 디렉터리의 YAML만 읽고 서브디렉터리의 Application을 생성하지 않는다.

### 단계 2: 적용

```bash
kubectl apply -f argocd/root-app.yaml
```
