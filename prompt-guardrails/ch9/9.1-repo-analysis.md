# 9.1 AI에게 레포 분석 시키기

## 실행 지침

### 단계 1: 저장소 구조 분석

notiflex-platform 저장소의 전체 구조를 분석한다:
- 디렉터리 트리
- 파일 개수
- 코드 라인 수 (언어별)

### 단계 2: 커밋 히스토리 분석

```bash
git log --oneline --all
```

각 커밋이 어떤 챕터에 해당하는지, 어떤 변화가 있었는지 정리한다.

### 단계 3: JOURNEY.md와 대조

`JOURNEY.md`의 진행 현황, 현재 리소스, 현재 버전 테이블을 읽어서
저장소 실제 상태와 일치하는지 확인한다. 불일치가 있으면 JOURNEY.md를 업데이트한다.
